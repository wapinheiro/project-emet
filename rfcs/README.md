# RFCs (Request for Comments)

This directory contains technical proposals for Project Emet.

## Purpose

RFCs are used to propose, discuss, and document significant changes and additions to Project Emet, including:

- New features and capabilities in the AIR specification
- Changes to the core semantics
- Tool and ecosystem proposals
- Integration specifications
- Formal verification methods

## RFC Process

1. **Draft**: Create a new RFC document with a descriptive name
2. **Discussion**: Share and gather feedback from the community
3. **Revision**: Iterate based on feedback
4. **Decision**: Accept, reject, or defer the proposal
5. **Implementation**: If accepted, implement according to the RFC

## RFC Template

When creating a new RFC, include:

- **Title**: Clear and concise
- **Author(s)**: Who is proposing this
- **Status**: Draft, Under Review, Accepted, Rejected, Implemented
- **Summary**: Brief overview of the proposal
- **Motivation**: Why this change is needed
- **Detailed Design**: Technical specification
- **Drawbacks**: Potential downsides
- **Alternatives**: Other approaches considered
- **Unresolved Questions**: Open issues for discussion
